# 热点监控工具 - 需求与架构文档

本项目旨在提供一个强大的自动化情报平台，用于全网发现最新热点信息、使用大语言模型进行真伪研判，并通过无边界词云画板和聚合邮件/Webhook及时发送精准通知。

## 核心功能特性

### 1. 动态无边界词云画板 (Infinite Word Cloud Canvas)
*   **交互体验**：响应式全屏 Web 界面，支持鼠标随意拖拽画板、滚轮无极缩放。
*   **词云生成**：通过抓取近期的全网热点新闻，由 AI 提炼热词。词与词之间拥有严谨的碰撞检测（GridSize = 8），防止重叠，同时每个球体根据数据量大小动态缩放半径。
*   **螺旋布局**：多个不同关键词的词云球体在画板上按螺旋形初始排布，确保互相不遮挡。
*   **刷新频率控制**：全局配置带有定制开发的**非线性时间轴吸附滑块**（支持 15 分钟至 1 周），可随时在 UI 端动态调整后端全局扫描的时间间隔。

### 2. 高精度热点通知系统 (Hotspot Alert Engine)
*   **独立任务隔离**：热点通知的关键词与全局词云相互独立。通知任务专注于“近 N 小时内的突发动态”。
*   **多平台并发巡查**：支持根据每个任务独立勾选检索源：DuckDuckGo（全网面）、小红书（潮流生活）、X/Twitter（国际科技）、Weibo（国内舆情）。
*   **全自动防风控**：底层爬虫配备严格的 `await sleep` 机制与并行度限制，有效防止触发目标网站的反爬验证。
*   **定制化频率**：每一个独立的报警任务都享有独立的非线性时间滑块（15min ~ 7days）。提供方便的“一键编辑”功能。

### 3. AI 智能研判与去重防骚扰 (AI Deduplication & Anti-Spam)
*   **内容去伪存真**：使用免费且强大的 SiliconFlow API（默认配置为 `DeepSeek-V3.2` 或 `Qwen` 旗舰模型），强迫 AI 必须输出包含“可信度打分 (0-100)”、“真伪判断”及纯简体中文的研判依据。
*   **MD5 数据指纹去重**：对于已经被抓取并发送过的历史新闻，系统会根据标题与来源生成唯一指纹，**绝对不会重复发送同一条新闻**。
*   **聚合发送 (Aggregation)**：告别单条信息轰炸。每次扫描周期结束时，系统会将收集到的多条最新动态打包汇总，以一封结构清晰的综合邮件（或一条长 Webhook）发送给用户。

### 4. 全局配置与状态流转
*   **UI 动态配置**：通过前端“系统设置”面板，用户可以随时随地修改全局默认 Webhook 地址、默认接收 Email、以及底层的 SMTP 发件服务器和授权码。配置会持久化至后端的 `.env` 文件中，保障敏感信息的安全。
*   **实时推送 (Socket.io)**：后台爬取到新数据或生成了新的词云形态后，会通过 WebSocket 毫秒级推送到前端画布，实现页面无缝热更新。

## 技术栈与架构设计

### 后端架构
*   **框架**：Express 5，原生支持现代异步路由处理。
*   **数据库与数据流**：Prisma ORM + SQLite。底层状态与核心业务逻辑依托外置到独立 `data/dev.db` 数据库文件。系统支持高度的数据隔离，每次扫描获取的真伪热点和词云模型都会作为情报资产，自动以 JSON 格式归档持久化到独立的 `data/hotspots/`、`data/wordclouds/` 和 `data/alerts/` 日志文件夹中。
*   **任务调度**：`node-cron` 用于执行精确的全局词云巡查与分发警报扫描任务。
*   **网络通信**：Axios + Cheerio 实现免 API Key 的定向开源搜索（如 DuckSearch 代理协议）。Socket.io 负责双工通讯。

### 前端架构
*   **框架**：React + Vite + TypeScript。
*   **可视化**：Echarts + `echarts-wordcloud` 用于构建带有碰撞检测的独立球体词云。
*   **交互**：原生 DOM 事件流（PointerDown, PointerMove, Wheel）手写无限画布系统。Lucide-React 图标库。

### 外部依赖与 AI
*   **大语言模型**：硅基流动 (SiliconFlow) 提供的 API 接口（兼容 OpenAI 标准格式）。
*   **推送通道**：`nodemailer` 处理 SMTP 标准邮件推送；支持自定义 Webhook payload 推送至微信机器人或飞书。

## 模块协同流转图 (Workflow)

1.  **触发层**：`node-cron` 根据滑动条设定的频率触发，或用户在 UI 侧主动点击“立即执行全网扫描”。
2.  **采集层**：鸭鸭搜索（DuckDuckGo）与站点约束（`site:xiaohongshu.com`）相结合，逐页采集最新的搜索片段。
3.  **清洗层**：过滤掉 SEO 垃圾数据，去重比对 `sourceHash`。
4.  **AI 层**：将清洗好的数万字网页片段送入大模型，按照严格 Prompt 解析并提取出 JSON 格式的可信度、标题、摘要。
5.  **分发层**：持久化存入 SQLite，并通过 NodeMailer 聚合发送纯中文报告。同步向连接着的所有前端客户端触发 `socket.emit`。
